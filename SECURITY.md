# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.0.x   | Yes       |

## Reporting a vulnerability

Do not open a public issue for secrets, bypasses, or exploitable behavior.

Send a private report to the project maintainer with:

- Affected component.
- Reproduction steps.
- Expected behavior.
- Actual behavior.
- Suggested fix if available.

## Project security rules

- Do not log raw secrets.
- Do not bypass rails.
- Do not bypass kill switches.
- Do not ship output that fails quality gates.
- Do not store production credentials in examples or tests.
